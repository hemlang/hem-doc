# Hemlock 版本管理

本文档描述了 Hemlock 的版本管理策略。

## 版本格式

Hemlock 使用**语义化版本控制**（SemVer）：

```
MAJOR.MINOR.PATCH
```

| 组件 | 何时递增 |
|------|---------|
| **MAJOR** | 语言语义、标准库 API 或二进制格式的不兼容变更 |
| **MINOR** | 新功能、向后兼容的新增内容 |
| **PATCH** | 错误修复、性能改进、文档更新 |

## 统一版本管理

所有 Hemlock 组件共享**同一个版本号**：

- **解释器** (`hemlock`)
- **编译器** (`hemlockc`)
- **LSP 服务器** (`hemlock --lsp`)
- **标准库** (`@stdlib/*`)

版本定义在 `include/version.h` 中：

```c
#define HEMLOCK_VERSION_MAJOR 1
#define HEMLOCK_VERSION_MINOR 8
#define HEMLOCK_VERSION_PATCH 7

#define HEMLOCK_VERSION "1.8.7"
```

### 检查版本

```bash
# 解释器版本
hemlock --version

# 编译器版本
hemlockc --version
```

## 兼容性保证

### 在同一 MAJOR 版本内

- 在 `1.x.0` 中可运行的源代码在 `1.x.y`（任何 patch）中同样可运行
- 在 `1.0.x` 中可运行的源代码在 `1.y.z`（任何 minor/patch）中同样可运行
- 编译后的 `.hmlb` 包在同一 MAJOR 版本内兼容
- 标准库 API 保持稳定（仅增加，不删除）

### 跨 MAJOR 版本

- 不兼容变更会记录在发布说明中
- 为重大变更提供迁移指南
- 废弃功能在移除前至少在一个 minor 版本中给出警告

## 二进制格式版本管理

Hemlock 对二进制格式使用单独的版本号：

| 格式 | 版本 | 位置 |
|------|------|------|
| `.hmlc`（AST 包） | `HMLC_VERSION` | `include/ast_serialize.h` |
| `.hmlb`（压缩包） | 与 HMLC 相同 | 使用 zlib 压缩 |
| `.hmlp`（打包的可执行文件） | Magic: `HMLP` | 自包含格式 |

二进制格式版本在序列化发生变更时独立递增。

## 标准库版本管理

标准库（`@stdlib/*`）**与主版本同步发布**：

```hemlock
// 始终使用与您的 Hemlock 安装捆绑的 stdlib
import { HashMap } from "@stdlib/collections";
import { sin, cos } from "@stdlib/math";
```

### 标准库兼容性

- 新模块可能在 MINOR 版本中添加
- 新函数可能在 MINOR 版本中添加到现有模块
- 函数签名在同一 MAJOR 版本内保持稳定
- 废弃函数在移除前会被标记并记录

## 发布流程

1. 在 `include/version.h` 中更新版本号
2. 更新变更日志
3. 运行完整测试套件（`make test-all`）
4. 在 git 中创建发布标签
5. 构建发布产物

## 检查兼容性

要验证您的代码是否适用于特定的 Hemlock 版本：

```bash
# 针对已安装的版本运行测试
make test

# 检查解释器和编译器之间的一致性
make parity
```
