# Глоссарий

Новичок в программировании или системных концепциях? Этот глоссарий объясняет термины, используемые в документации Hemlock, простым языком.

---

## A

### Выделение памяти (Allocate / Allocation)
**Что это значит:** Запрос у компьютера участка памяти для использования.

**Аналогия:** Как взять книгу в библиотеке — вы берёте на время пространство, которое нужно вернуть позже.

**В Hemlock:**
```hemlock
let space = alloc(100);  // "I need 100 bytes of memory, please"
// ... use it ...
free(space);             // "I'm done, you can have it back"
```

### Массив (Array)
**Что это значит:** Список значений, хранящихся вместе, с доступом по позиции (индексу).

**Аналогия:** Как ряд почтовых ящиков, пронумерованных 0, 1, 2, 3... Вы можете положить что-то в ящик #2 и позже забрать из ящика #2.

**В Hemlock:**
```hemlock
let colors = ["red", "green", "blue"];
print(colors[0]);  // "red" - first item is at position 0
print(colors[2]);  // "blue" - third item is at position 2
```

### Async / Асинхронный (Asynchronous)
**Что это значит:** Код, который может выполняться «в фоновом режиме», пока другой код продолжает работать. В Hemlock асинхронный код действительно выполняется на отдельных ядрах CPU одновременно.

**Аналогия:** Как готовить несколько блюд одновременно — вы ставите рис, и пока он варится, нарезаете овощи. Оба процесса происходят одновременно.

**В Hemlock:**
```hemlock
async fn slow_task(): i32 {
    // This can run on another CPU core
    return 42;
}

let task = spawn(slow_task);  // Start it running
// ... do other stuff while it runs ...
let result = join(task);      // Wait for it to finish, get result
```

---

## B

### Булево значение (Boolean / Bool)
**Что это значит:** Значение, которое может быть только `true` или `false`. Ничего другого.

**Названо в честь:** Джорджа Буля, математика, изучавшего логику истины/лжи.

**В Hemlock:**
```hemlock
let is_raining = true;
let has_umbrella = false;

if (is_raining && !has_umbrella) {
    print("You'll get wet!");
}
```

### Проверка границ (Bounds Checking)
**Что это значит:** Автоматическая проверка того, что вы не пытаетесь получить доступ к памяти за пределами выделенного. Предотвращает сбои и уязвимости безопасности.

**Аналогия:** Как библиотекарь, который проверяет, что запрашиваемая вами книга действительно существует, прежде чем пытаться её найти.

**В Hemlock:**
```hemlock
let buf = buffer(10);  // 10 slots, numbered 0-9
buf[5] = 42;           // OK - slot 5 exists
buf[100] = 42;         // ERROR! Hemlock stops you - slot 100 doesn't exist
```

### Буфер (Buffer)
**Что это значит:** Безопасный контейнер для сырых байтов с известным размером. Hemlock проверяет, что вы не читаете и не записываете за его границы.

**Аналогия:** Как сейф с определённым количеством отделений. Вы можете использовать любое отделение, но не можете открыть отделение #50, если в сейфе только 10.

**В Hemlock:**
```hemlock
let data = buffer(64);   // 64 bytes of safe storage
data[0] = 65;            // Put 65 in the first byte
print(data.length);      // 64 - you can check its size
free(data);              // Clean up when done
```

## C

### Замыкание (Closure)
**Что это значит:** Функция, которая «помнит» переменные из того места, где она была создана, даже после того, как тот код завершился.

**Аналогия:** Как записка, на которой написано «прибавь 5 к любому числу, которое мне дадут» — «5» встроено в записку.

**В Hemlock:**
```hemlock
fn make_adder(amount) {
    return fn(x) {
        return x + amount;  // 'amount' is remembered!
    };
}

let add_five = make_adder(5);
print(add_five(10));  // 15 - it remembered that amount=5
```

### Приведение типов (Coercion / Type Coercion)
**Что это значит:** Автоматическое преобразование значения из одного типа в другой при необходимости.

**Пример:** Когда вы складываете целое число и десятичное, целое число автоматически преобразуется в десятичное.

**В Hemlock:**
```hemlock
let whole: i32 = 5;
let decimal: f64 = 2.5;
let result = whole + decimal;  // 'whole' becomes 5.0, then adds to 2.5
print(result);  // 7.5
```

### Компиляция / Компилятор (Compile / Compiler)
**Что это значит:** Перевод вашего кода в программу, которую компьютер может выполнить напрямую. Компилятор (`hemlockc`) читает ваш файл `.hml` и создаёт исполняемый файл.

**Аналогия:** Как перевод книги с английского на испанский — содержание то же, но теперь испаноговорящие могут её читать.

**В Hemlock:**
```bash
hemlockc myprogram.hml -o myprogram   # Translate to executable
./myprogram                            # Run the executable
```

### Конкурентность (Concurrency)
**Что это значит:** Несколько вещей происходят в перекрывающиеся промежутки времени. В Hemlock это означает реальное параллельное выполнение на нескольких ядрах CPU.

**Аналогия:** Два повара готовят разные блюда одновременно на одной кухне.

## D

### Отложенное выполнение (Defer)
**Что это значит:** Запланировать что-то на потом, когда текущая функция завершится. Полезно для очистки ресурсов.

**Аналогия:** Как сказать себе «когда буду уходить, выключу свет» — вы задаёте напоминание сейчас, оно выполнится потом.

**В Hemlock:**
```hemlock
fn process_file() {
    let f = open("data.txt", "r");
    defer f.close();  // "Close this file when I'm done here"

    // ... lots of code ...
    // Even if there's an error, f.close() will run
}
```

### Утиная типизация (Duck Typing)
**Что это значит:** Если оно выглядит как утка и крякает как утка, обращайтесь с ним как с уткой. В коде: если объект имеет нужные вам поля/методы, используйте его — не беспокойтесь о его официальном «типе».

**Названо в честь:** Утиного теста — формы рассуждения.

**В Hemlock:**
```hemlock
define Printable {
    name: string
}

fn greet(thing: Printable) {
    print("Hello, " + thing.name);
}

// Any object with a 'name' field works!
greet({ name: "Alice" });
greet({ name: "Bob", age: 30 });  // Extra fields are OK
```

## E

### Выражение (Expression)
**Что это значит:** Код, который производит значение. Может использоваться везде, где ожидается значение.

**Примеры:** `42`, `x + y`, `get_name()`, `true && false`

### Перечисление (Enum / Enumeration)
**Что это значит:** Тип с фиксированным набором возможных значений, каждое с именем.

**Аналогия:** Как выпадающее меню — вы можете выбрать только из перечисленных вариантов.

**В Hemlock:**
```hemlock
enum Status {
    PENDING,
    APPROVED,
    REJECTED
}

let my_status = Status.APPROVED;

if (my_status == Status.REJECTED) {
    print("Sorry!");
}
```

## F

### Число с плавающей точкой (Float / Floating-Point)
**Что это значит:** Число с десятичной точкой. Называется «плавающей точкой», потому что десятичная точка может быть в разных позициях.

**В Hemlock:**
```hemlock
let pi = 3.14159;      // f64 - 64-bit float (default)
let half: f32 = 0.5;   // f32 - 32-bit float (smaller, less precise)
```

### Освобождение памяти (Free)
**Что это значит:** Вернуть память, которую вы больше не используете, обратно системе, чтобы её можно было использовать повторно.

**Аналогия:** Вернуть книгу в библиотеку, чтобы другие могли её взять.

**В Hemlock:**
```hemlock
let data = alloc(100);  // Borrow 100 bytes
// ... use data ...
free(data);             // Return it - REQUIRED!
```

### Функция (Function)
**Что это значит:** Повторно используемый блок кода, который принимает входные данные (параметры) и может производить выходное значение (возвращаемое значение).

**Аналогия:** Как рецепт — дайте ему ингредиенты (входы), следуйте шагам, получите блюдо (выход).

**В Hemlock:**
```hemlock
fn add(a, b) {
    return a + b;
}

let result = add(3, 4);  // result is 7
```

## G

### Сборка мусора (Garbage Collection / GC)
**Что это значит:** Автоматическая очистка памяти. Среда выполнения периодически находит неиспользуемую память и освобождает её за вас.

**Почему в Hemlock её нет:** GC может вызывать непредсказуемые паузы. Hemlock предпочитает явный контроль — вы решаете, когда освобождать память.

**Примечание:** Большинство типов Hemlock (строки, массивы, объекты) АВТОМАТИЧЕСКИ очищаются при выходе из области видимости. Только сырой `ptr` из `alloc()` требует ручного `free()`.

## H

### Куча (Heap)
**Что это значит:** Область памяти для данных, которые должны существовать дольше текущей функции. Вы явно выделяете и освобождаете память в куче.

**В противоположность:** Стек (автоматическое, временное хранение для локальных переменных)

**В Hemlock:**
```hemlock
let ptr = alloc(100);  // This goes on the heap
// ... use it ...
free(ptr);             // You clean up the heap yourself
```

## I

### Индекс (Index)
**Что это значит:** Позиция элемента в массиве или строке. Начинается с 0 в Hemlock.

**В Hemlock:**
```hemlock
let letters = ["a", "b", "c"];
//             [0]  [1]  [2]   <- indices

print(letters[0]);  // "a" - first item
print(letters[2]);  // "c" - third item
```

### Целое число (Integer)
**Что это значит:** Целое число без десятичной точки. Может быть положительным, отрицательным или нулём.

**В Hemlock:**
```hemlock
let small = 42;       // i32 - fits in 32 bits
let big = 5000000000; // i64 - needs 64 bits (auto-detected)
let tiny: i8 = 100;   // i8 - explicitly 8 bits
```

### Интерпретатор (Interpreter)
**Что это значит:** Программа, которая читает ваш код и выполняет его напрямую, строка за строкой.

**В противоположность:** Компилятор (сначала переводит код, потом выполняет перевод)

**В Hemlock:**
```bash
./hemlock script.hml   # Interpreter runs your code directly
```

## L

### Литерал (Literal)
**Что это значит:** Значение, записанное непосредственно в вашем коде, а не вычисленное.

**Примеры:**
```hemlock
42              // integer literal
3.14            // float literal
"hello"         // string literal
true            // boolean literal
[1, 2, 3]       // array literal
{ x: 10 }       // object literal
```

## M

### Утечка памяти (Memory Leak)
**Что это значит:** Забыть освободить выделенную память. Память остаётся зарезервированной, но неиспользуемой, расходуя ресурсы впустую.

**Аналогия:** Взять книги в библиотеке и никогда их не вернуть. В конце концов в библиотеке закончатся книги.

**В Hemlock:**
```hemlock
fn leaky() {
    let ptr = alloc(1000);
    // Oops! Forgot to free(ptr)
    // Those 1000 bytes are lost until program exits
}
```

### Метод (Method)
**Что это значит:** Функция, прикреплённая к объекту или типу.

**В Hemlock:**
```hemlock
let text = "hello";
let upper = text.to_upper();  // to_upper() is a method on strings
print(upper);  // "HELLO"
```

### Мьютекс (Mutex)
**Что это значит:** Блокировка, обеспечивающая доступ к чему-либо только одному потоку за раз. Предотвращает повреждение данных, когда несколько потоков работают с общими данными.

**Аналогия:** Как замок в ванной — только один человек может пользоваться ей одновременно.

### Null
**Что это значит:** Специальное значение, означающее «ничего» или «нет значения».

**В Hemlock:**
```hemlock
let maybe_name = null;

if (maybe_name == null) {
    print("No name provided");
}
```

## O

### Объект (Object)
**Что это значит:** Коллекция именованных значений (полей/свойств), сгруппированных вместе.

**В Hemlock:**
```hemlock
let person = {
    name: "Alice",
    age: 30,
    city: "NYC"
};

print(person.name);  // "Alice"
print(person.age);   // 30
```

## P

### Параметр (Parameter)
**Что это значит:** Переменная, которую функция ожидает получить при вызове.

**Также называется:** Аргумент (технически, параметр — в определении, аргумент — при вызове)

**В Hemlock:**
```hemlock
fn greet(name, times) {   // 'name' and 'times' are parameters
    // ...
}

greet("Alice", 3);        // "Alice" and 3 are arguments
```

### Указатель (Pointer)
**Что это значит:** Значение, содержащее адрес памяти — оно «указывает» на то, где хранятся данные.

**Аналогия:** Как почтовый адрес. Адрес — это не дом, он говорит вам, где найти дом.

**В Hemlock:**
```hemlock
let ptr = alloc(100);  // ptr holds the address of 100 bytes
// ptr doesn't contain the data - it points to where the data lives
free(ptr);
```

### Примитив (Primitive)
**Что это значит:** Базовый, встроенный тип, который не состоит из других типов.

**В Hemlock:** `i8`, `i16`, `i32`, `i64`, `u8`, `u16`, `u32`, `u64`, `f32`, `f64`, `bool`, `rune`, `null`

## R

### Подсчёт ссылок (Reference Counting / Refcounting)
**Что это значит:** Отслеживание того, сколько вещей используют определённый фрагмент данных. Когда ничто больше не использует его, данные автоматически очищаются.

**В Hemlock:** Строки, массивы, объекты и буферы используют подсчёт ссылок внутренне. Вы этого не видите, но это предотвращает утечки памяти для большинства распространённых типов.

### Возвращаемое значение (Return Value)
**Что это значит:** Значение, которое функция отправляет обратно при завершении.

**В Hemlock:**
```hemlock
fn double(x) {
    return x * 2;  // This is the return value
}

let result = double(5);  // result gets the return value: 10
```

### Руна (Rune)
**Что это значит:** Один символ Unicode (кодовая точка). Может представлять любой символ, включая эмодзи.

**Почему «руна»?** Термин пришёл из Go. Он подчёркивает, что это полноценный символ, а не просто байт.

**В Hemlock:**
```hemlock
let letter = 'A';
let emoji = '🚀';
let code: i32 = letter;  // 65 - the Unicode codepoint
```

### Время выполнения (Runtime)
**Что это значит:** Время, когда ваша программа действительно выполняется (в отличие от «времени компиляции», когда она переводится).

**Также:** Вспомогательный код, который работает вместе с вашей программой (например, аллокатор памяти).

## S

### Область видимости (Scope)
**Что это значит:** Область кода, где переменная существует и может быть использована.

**В Hemlock:**
```hemlock
let outer = 1;              // Lives in outer scope

if (true) {
    let inner = 2;          // Lives only inside this block
    print(outer);           // OK - can see outer scope
    print(inner);           // OK - we're inside its scope
}

print(outer);               // OK
// print(inner);            // ERROR - inner doesn't exist here
```

### Стек (Stack)
**Что это значит:** Память для временных, кратковременных данных. Управляется автоматически — когда функция возвращается, её пространство стека освобождается.

**В противоположность:** Куча (более долгоживущая, управляемая вручную)

### Инструкция (Statement)
**Что это значит:** Одна команда или указание. Инструкции ДЕЛАЮТ вещи; выражения ПРОИЗВОДЯТ значения.

**Примеры:** `let x = 5;`, `print("hi");`, `if (x > 0) { ... }`

### Строка (String)
**Что это значит:** Последовательность текстовых символов.

**В Hemlock:**
```hemlock
let greeting = "Hello, World!";
print(greeting.length);    // 13 characters
print(greeting[0]);        // "H" - first character
```

### Структурная типизация (Structural Typing)
**Что это значит:** Совместимость типов, основанная на структуре (какие поля/методы существуют), а не на имени. То же, что и «утиная типизация».

## T

### Поток (Thread)
**Что это значит:** Отдельный путь выполнения. Несколько потоков могут работать одновременно на разных ядрах CPU.

**В Hemlock:** `spawn()` создаёт новый поток.

### Тип (Type)
**Что это значит:** Вид данных, который представляет значение. Определяет, какие операции допустимы.

**В Hemlock:**
```hemlock
let x = 42;              // type: i32
let name = "Alice";      // type: string
let nums = [1, 2, 3];    // type: array

print(typeof(x));        // "i32"
print(typeof(name));     // "string"
```

### Аннотация типа (Type Annotation)
**Что это значит:** Явное объявление того, какой тип должна иметь переменная.

**В Hemlock:**
```hemlock
let x: i32 = 42;         // x must be an i32
let name: string = "hi"; // name must be a string

fn add(a: i32, b: i32): i32 {  // parameters and return type annotated
    return a + b;
}
```

## U

### UTF-8
**Что это значит:** Способ кодирования текста, поддерживающий все языки мира и эмодзи. Каждый символ может занимать 1-4 байта.

**В Hemlock:** Все строки имеют кодировку UTF-8.

```hemlock
let text = "Hello, 世界! 🌍";  // Mix of ASCII, Chinese, emoji - all work
```

## V

### Переменная (Variable)
**Что это значит:** Именованное место хранения, содержащее значение.

**В Hemlock:**
```hemlock
let count = 0;    // Create variable 'count', store 0
count = count + 1; // Update it to 1

```

## Краткий справочник: Какой тип использовать?

| Ситуация | Используйте это | Почему |
|----------|-----------------|--------|
| Просто нужно число | `let x = 42;` | Hemlock выберет правильный тип |
| Подсчёт чего-либо | `i32` | Достаточно велик для большинства подсчётов |
| Огромные числа | `i64` | Когда i32 недостаточно |
| Байты (0-255) | `u8` | Файлы, сетевые данные |
| Десятичные | `f64` | Точная десятичная математика |
| Значения Да/Нет | `bool` | Только `true` или `false` |
| Текст | `string` | Любое текстовое содержимое |
| Один символ | `rune` | Одна буква/эмодзи |
| Список вещей | `array` | Упорядоченная коллекция |
| Именованные поля | `object` | Группировка связанных данных |
| Сырая память | `buffer` | Безопасное хранение байтов |
| FFI/системная работа | `ptr` | Продвинутое, ручное управление памятью |
