## Краткое содержание

Система аннотаций Hemlock предоставляет надёжную основу для добавления подсказок и директив компилятора. Это предложение расширяет текущую инфраструктуру аннотаций **15 новыми вспомогательными аннотациями компилятора**, организованными в пять категорий:

- **Подсказки оптимизации** (7 аннотаций)
- **Управление памятью** (3 аннотации)
- **Управление генерацией кода** (2 аннотации)
- **Проверка ошибок** (2 аннотации)
- **FFI/Интероп** (1 аннотация)

## Анализ текущего состояния

### Реализованные аннотации

```c
// Аннотации безопасности (для проверки памяти Tricycle)
@safe       // Функция безопасна по памяти
@unsafe     // Функция содержит небезопасные операции
@trusted    // Функция доверена несмотря на небезопасные операции

// Подсказки оптимизации компилятора (РЕАЛИЗОВАНО в v1.9.0)
@inline     // Предложить встраивание функции
@noinline   // Предотвратить встраивание
@cold       // Функция выполняется редко
@hot        // Функция выполняется часто
@pure       // Функция без побочных эффектов

// Другие аннотации
@deprecated      // Отметить как устаревшее
@test, @skip     // Аннотации тестирующего фреймворка
@author, @since  // Документационные аннотации
```

**Обновление (v1.9.0):** Основные аннотации уровня функций (`@inline`, `@noinline`, `@hot`, `@cold`, `@pure`, `@const`, `@flatten`, `@optimize`, `@warn_unused`, `@section`) полностью реализованы в бэкенде компилятора. Остальные предложения (аннотации циклов, памяти) в Фазах 3-5 ниже ещё не реализованы.

## Предложенные аннотации

### Категория 1: Подсказки оптимизации

- `@unroll(count?)` - Предложение развёртки цикла
- `@simd` / `@nosimd` - Включение/отключение SIMD-векторизации
- `@likely` / `@unlikely` - Подсказки предсказания ветвлений
- `@const` - Функция зависит только от аргументов (сильнее `@pure`)
- `@tail_call` - Запрос оптимизации хвостовых вызовов
- `@flatten` - Встроить все вызовы внутри функции
- `@optimize(level)` - Переопределить уровень оптимизации для функции

### Категория 2: Управление памятью

- `@stack` - Размещение на стеке вместо кучи
- `@noalias` - Указатель не является псевдонимом для других указателей
- `@aligned(bytes)` - Требования к выравниванию памяти

### Категория 3: Управление генерацией кода

- `@extern(name?, abi?)` - Внешняя линковка или экспорт FFI
- `@section(name)` - Размещение в определённой секции ELF/Mach-O

### Категория 4: Проверка ошибок

- `@bounds_check` / `@no_bounds_check` - Переопределение глобальной политики проверки границ
- `@warn_unused` - Предупреждение при игнорировании возвращаемого значения

### Категория 5: FFI/Интероп

- `@packed` - Упакованная структура без выравнивания (для C-интеропа)

## Полная справочная таблица аннотаций

| Аннотация | Цель | Аргументы | Описание | Атрибут C |
|-----------|------|-----------|----------|-----------|
| `@inline` | fn | 0 | Форсировать встраивание | `always_inline` |
| `@noinline` | fn | 0 | Предотвратить встраивание | `noinline` |
| `@cold` | fn | 0 | Редко выполняемая | `cold` |
| `@hot` | fn | 0 | Часто выполняемая | `hot` |
| `@pure` | fn | 0 | Без побочных эффектов, может читать глобальные | `pure` |
| `@const` | fn | 0 | Без побочных эффектов, без чтения глобальных | `const` |
| `@flatten` | fn | 0 | Встроить все вызовы внутри функции | `flatten` |
| `@tail_call` | fn | 0 | Запрос оптимизации хвостовых вызовов | Пользовательский |
| `@optimize(level)` | fn | 1 | Переопределить уровень оптимизации | `optimize("OX")` |
| `@unroll(factor?)` | loop | 0-1 | Подсказка развёртки цикла | `#pragma unroll` |
| `@simd` | fn, loop | 0 | Включить SIMD-векторизацию | `#pragma omp simd` |
| `@nosimd` | fn, loop | 0 | Отключить SIMD | Пользовательский |
| `@likely` | if | 0 | Ветвление вероятно выполняется | `__builtin_expect` |
| `@unlikely` | if | 0 | Ветвление маловероятно | `__builtin_expect` |
| `@stack` | let | 0 | Размещение на стеке | Пользовательский |
| `@noalias` | param | 0 | Нет псевдонимов указателей | `noalias` |
| `@aligned(N)` | let, fn | 1 | Выравнивание памяти | `aligned(N)` |
| `@extern(name?, abi?)` | fn | 0-2 | Внешняя линковка | `extern "C"` |
| `@section(name)` | fn, let | 1 | Размещение в определённой секции | `section("X")` |
| `@bounds_check` | fn | 0 | Форсировать проверку границ | Пользовательский |
| `@no_bounds_check` | fn | 0 | Отключить проверку границ | Пользовательский |
| `@warn_unused` | fn | 0 | Предупреждение о неиспользованном возврате | `warn_unused_result` |
| `@packed` | define | 0 | Без выравнивания структуры | `packed` |
