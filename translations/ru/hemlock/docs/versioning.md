# Версионирование Hemlock

Этот документ описывает стратегию версионирования Hemlock.

## Формат версии

Hemlock использует **семантическое версионирование** (SemVer):

```
MAJOR.MINOR.PATCH
```

| Компонент | Когда увеличивать |
|-----------|-------------------|
| **MAJOR** | Несовместимые изменения в семантике языка, API стандартной библиотеки или бинарных форматах |
| **MINOR** | Новые возможности, обратно совместимые дополнения |
| **PATCH** | Исправления ошибок, улучшения производительности, документация |

## Единое версионирование

Все компоненты Hemlock используют **единый номер версии**:

- **Интерпретатор** (`hemlock`)
- **Компилятор** (`hemlockc`)
- **LSP-сервер** (`hemlock --lsp`)
- **Стандартная библиотека** (`@stdlib/*`)

Версия определяется в `include/version.h`:

```c
#define HEMLOCK_VERSION_MAJOR 1
#define HEMLOCK_VERSION_MINOR 8
#define HEMLOCK_VERSION_PATCH 7

#define HEMLOCK_VERSION "1.8.7"
```

### Проверка версий

```bash
# Версия интерпретатора
hemlock --version

# Версия компилятора
hemlockc --version
```

## Гарантии совместимости

### В пределах версии MAJOR

- Исходный код, работающий в `1.x.0`, будет работать в `1.x.y` (любой patch)
- Исходный код, работающий в `1.0.x`, будет работать в `1.y.z` (любой minor/patch)
- Скомпилированные пакеты `.hmlb` совместимы в пределах одной версии MAJOR
- API стандартной библиотеки стабильны (только добавления, без удалений)

### Между версиями MAJOR

- Несовместимые изменения документируются в примечаниях к выпуску
- Руководства по миграции предоставляются для значительных изменений
- Устаревшие функции сопровождаются предупреждением как минимум в одном минорном выпуске перед удалением

## Версионирование бинарных форматов

Hemlock использует отдельные номера версий для бинарных форматов:

| Формат | Версия | Расположение |
|--------|--------|--------------|
| `.hmlc` (пакет AST) | `HMLC_VERSION` | `include/ast_serialize.h` |
| `.hmlb` (сжатый пакет) | Совпадает с HMLC | Использует сжатие zlib |
| `.hmlp` (упакованный исполняемый файл) | Magic: `HMLP` | Автономный формат |

Версии бинарных форматов увеличиваются независимо при изменении сериализации.

## Версионирование стандартной библиотеки

Стандартная библиотека (`@stdlib/*`) версионируется **вместе с основным выпуском**:

```hemlock
// Всегда использует stdlib, поставляемую с вашей установкой Hemlock
import { HashMap } from "@stdlib/collections";
import { sin, cos } from "@stdlib/math";
```

### Совместимость stdlib

- Новые модули могут быть добавлены в минорных выпусках
- Новые функции могут быть добавлены в существующие модули в минорных выпусках
- Сигнатуры функций стабильны в пределах версии MAJOR
- Устаревшие функции помечаются и документируются перед удалением

## Процесс выпуска

1. Обновление версии в `include/version.h`
2. Обновление журнала изменений
3. Запуск полного набора тестов (`make test-all`)
4. Создание тега выпуска в git
5. Сборка артефактов выпуска

## Проверка совместимости

Чтобы убедиться, что ваш код работает с определённой версией Hemlock:

```bash
# Запуск тестов для установленной версии
make test

# Проверка паритета между интерпретатором и компилятором
make parity
```

## Будущее: манифесты проектов

В будущем выпуске могут быть введены необязательные манифесты проектов для ограничений версий:

```hemlock
// Гипотетический project.hml
define Project {
    name: "my-app",
    version: "1.0.0",
    hemlock: ">=1.1.0"
}
```

Это ещё не реализовано, но входит в план развития.
