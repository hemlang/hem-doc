## Обзор

Успешно реализованы вспомогательные аннотации компилятора для Hemlock, позволяющие разработчикам предоставлять явные подсказки оптимизации для GCC/Clang через генерируемые атрибуты C. Это расширяет существующую инфраструктуру аннотаций 13 новыми типами аннотаций.

## Что было реализовано

### Фаза 1: Существующие аннотации функций (Commit: 0754a49)

Подключены 5 аннотаций, которые существовали в спецификации, но не использовались компилятором:

| Аннотация | Атрибут C | Назначение |
|-----------|-----------|------------|
| `@inline` | `__attribute__((always_inline))` | Принудительное встраивание функции |
| `@noinline` | `__attribute__((noinline))` | Запрет встраивания функции |
| `@hot` | `__attribute__((hot))` | Часто исполняемый код |
| `@cold` | `__attribute__((cold))` | Редко исполняемый код |
| `@pure` | `__attribute__((pure))` | Без побочных эффектов, может читать глобальные переменные |

**Пример:**
```hemlock
@inline
@hot
fn critical_path(n: i32): i32 => n * n;
```

**Генерируемый C:**
```c
__attribute__((always_inline)) __attribute__((hot))
HmlValue hml_fn_critical_path(HmlClosureEnv *_closure_env, HmlValue n) { ... }
```

### Фаза 2: @const и @flatten (Commit: 4f28796)

Добавлены 2 новые аннотации для более строгой чистоты и агрессивного встраивания:

| Аннотация | Атрибут C | Назначение |
|-----------|-----------|------------|
| `@const` | `__attribute__((const))` | Строже, чем @pure - без чтения глобальных переменных |
| `@flatten` | `__attribute__((flatten))` | Встроить ВСЕ вызовы внутри функции |

**Ключевое исправление:** Разрешён конфликт ключевого слова `const` путём добавления `TOK_CONST` в список контекстных идентификаторов.

**Пример:**
```hemlock
@const
fn square(x: i32): i32 => x * x;

@flatten
fn process(n: i32): i32 {
    let a = helper1(n);
    let b = helper2(a);
    return helper3(b);  // All helpers inlined
}
```

### Фаза 3: @optimize(level) (Commit: f538723)

Добавлена параметризованная аннотация для управления оптимизацией на уровне функции:

| Аннотация | Аргументы | Атрибут C | Назначение |
|-----------|-----------|-----------|------------|
| `@optimize(level)` | "0", "1", "2", "3", "s", "fast" | `__attribute__((optimize("-OX")))` | Переопределить уровень оптимизации |

**Пример:**
```hemlock
@optimize("3")     // Aggressive optimizations
fn matrix_multiply(a: i32, b: i32): i32 { ... }

@optimize("s")     // Optimize for size
fn error_handler(): void { ... }

@optimize("0")     // No optimization (debugging)
fn debug_function(): void { ... }
```

**Генерируемый C:**
```c
__attribute__((optimize("-O3"))) HmlValue hml_fn_matrix_multiply(...)
__attribute__((optimize("-Os"))) HmlValue hml_fn_error_handler(...)
__attribute__((optimize("-O0"))) HmlValue hml_fn_debug_function(...)
```

### Фаза 4: @warn_unused (Commit: 80e435b)

Добавлена аннотация для обнаружения ошибок, когда игнорируются важные возвращаемые значения:

| Аннотация | Атрибут C | Назначение |
|-----------|-----------|------------|
| `@warn_unused` | `__attribute__((warn_unused_result))` | Предупреждать, если возвращаемое значение проигнорировано |

**Пример:**
```hemlock
@warn_unused
fn allocate_memory(size: i32): ptr {
    return alloc(size);
}

// OK: Return value used
let p = allocate_memory(1024);

// WARN: Return value ignored (compiler warning)
allocate_memory(1024);
```

### Фазы 5-8: Аннотации памяти/FFI (Commit: 79a8b92)

Добавлены 3 аннотации для управления расположением в памяти и FFI:

| Аннотация | Цель | Аргументы | Статус | Назначение |
|-----------|------|-----------|--------|------------|
| `@section(name)` | Функции/Переменные | 1 строка | ✅ Реализовано | Пользовательское размещение в ELF-секции |
| `@aligned(N)` | Переменные | 1 число | ⚠️ Только спецификация | Выравнивание памяти |
| `@packed` | Структуры (define) | Нет | ⚠️ Только спецификация | Без заполнения структуры |

**Пример @section:**
```hemlock
@section(".text.hot")
@hot
fn critical_init(): void { ... }

@section(".text.cold")
@cold
fn error_handler(): void { ... }
```

**Генерируемый C:**
```c
__attribute__((hot)) __attribute__((section(".text.hot")))
HmlValue hml_fn_critical_init(...)

__attribute__((cold)) __attribute__((section(".text.cold")))
HmlValue hml_fn_error_handler(...)
```

## Архитектура

### Конвейер аннотаций

```
Hemlock Source Code
        ↓
    [Parser] - Разбирает @annotations, создаёт узлы AST
        ↓
  [Validator] - Проверяет цели, количество аргументов
        ↓
   [Resolver] - Сохраняет аннотации для семантических проверок
        ↓
   [Codegen] - Генерирует GCC/Clang __attribute__((...))
        ↓
  Generated C Code
        ↓
   [GCC/Clang] - Применяет реальные оптимизации
        ↓
  Optimized Binary
```

### Ключевые детали реализации

**1. Хранение аннотаций**
- Аннотации прикрепляются к узлам инструкций AST
- Парсер извлекает из синтаксиса `@name` или `@name(args)`
- Валидируются по таблице `AnnotationSpec`

**2. Интеграция с Codegen**
- Добавлен вспомогательный метод `codegen_emit_function_attributes()`
- Изменён `codegen_function_decl()` для принятия аннотаций
- Аннотации извлекаются из узлов `STMT_LET` и `STMT_EXPORT`
- Генерируемые атрибуты помещаются перед сигнатурой функции

**3. Поддержка модулей**
- Функции модулей получают аннотации через `codegen_module_funcs()`
- Аннотации извлекаются как из экспортируемых, так и из внутренних функций
- Предварительные объявления опускают атрибуты (только на реализации)

## Влияние на производительность

Аннотации предоставляют подсказки оптимизации, но не гарантируют конкретное поведение:

- **@inline**: GCC может не встроить, если слишком сложно
- **@hot/@cold**: Влияет на предсказание ветвлений и расположение кода
- **@optimize**: Переопределяет глобальный флаг `-O` для конкретных функций
- **@section**: Пользовательское размещение может улучшить локальность кэша
