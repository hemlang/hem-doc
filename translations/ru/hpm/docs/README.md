# Документация hpm

Добро пожаловать в документацию hpm (Hemlock Package Manager). hpm — это официальный менеджер пакетов для языка программирования [Hemlock](https://github.com/hemlang/hemlock).

## Обзор

hpm использует GitHub в качестве реестра пакетов, где пакеты идентифицируются по пути их GitHub-репозитория (например, `hemlang/sprout`). Это означает:

- **Нет центрального реестра** — пакеты находятся в GitHub-репозиториях
- **Теги версий** — релизы являются Git-тегами (например, `v1.0.0`)
- **Публикация — это просто git** — отправьте тег для публикации новой версии

## Краткий справочник

### Основные команды

```bash
hpm init                              # Создать новый package.json
hpm install                           # Установить все зависимости
hpm install owner/repo                # Добавить и установить пакет
hpm install owner/repo@^1.0.0        # Установить с ограничением версии
hpm uninstall owner/repo              # Удалить пакет
hpm update                            # Обновить все пакеты
hpm list                              # Показать установленные пакеты
hpm run <script>                      # Запустить скрипт пакета
```

### Идентификация пакетов

Пакеты используют формат GitHub `owner/repo`:

```
hemlang/sprout          # Веб-фреймворк
hemlang/json            # Утилиты для JSON
alice/http-client       # HTTP-клиент
```

### Ограничения версий

| Синтаксис | Значение |
|-----------|----------|
| `1.0.0` | Точная версия |
| `^1.2.3` | Совместимые (>=1.2.3 <2.0.0) |
| `~1.2.3` | Патч-обновления (>=1.2.3 <1.3.0) |
| `>=1.0.0` | Минимум 1.0.0 |
| `*` | Любая версия |

## Получение помощи

- Используйте `hpm --help` для справки в командной строке
- Используйте `hpm <command> --help` для справки по конкретной команде
- Сообщайте о проблемах на [github.com/hemlang/hpm/issues](https://github.com/hemlang/hpm/issues)
