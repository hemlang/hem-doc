## エグゼクティブサマリー

Hemlockのアノテーションシステムは、コンパイラのヒントやディレクティブを追加するための堅固な基盤を提供します。この提案は、現在のアノテーションインフラを5つのカテゴリに整理された**15個の新しいコンパイラヘルパーアノテーション**で拡張します：

- **最適化ヒント**（7アノテーション）
- **メモリ管理**（3アノテーション）
- **コード生成制御**（2アノテーション）
- **エラーチェック**（2アノテーション）
- **FFI/相互運用**（1アノテーション）

これらのアノテーションにより、開発者はインタプリタとの後方互換性を維持しながら、コンパイラ（`hemlockc`）に明示的なガイダンスを提供できます。

## 現状分析

### 1. アノテーションインフラ

アノテーションシステムは3つの主要コンポーネントで完全に実装されています：

**パーサー**（`src/frontend/parser/statements.c`）：
- `@name`および`@name(args...)`構文をパース
- 位置引数と名前付き引数をサポート
- アノテーションを宣言（let、const、define、enum）に付加

**バリデータ**（`src/frontend/annotations.c`）：
- アノテーションのターゲット（関数、型、変数など）を検証
- 引数の数と型をチェック
- 不明または重複するアノテーションに対して警告

**リゾルバ**（`src/frontend/resolver.c`）：
- 変数定義と共にアノテーションを保存
- セマンティック分析中のアノテーション検索を可能に
- 変数使用時の`@deprecated`警告を提供

### 2. 現在実装されているアノテーション

```c
// 安全性アノテーション（Tricycleメモリチェッカー用）
@safe       // 関数はメモリセーフ
@unsafe     // 関数はアンセーフな操作を含む
@trusted    // アンセーフな操作にもかかわらず関数は信頼される

// コンパイラ最適化ヒント（v1.9.0で実装済み）
@inline     // この関数のインライン化を提案
@noinline   // この関数のインライン化を防止
@cold       // まれに実行される
@hot        // 頻繁に実行される
@pure       // 副作用なし

// その他のアノテーション
@deprecated      // 非推奨としてマーク（オプションのメッセージ付き）
@test, @skip     // テストフレームワークアノテーション
@author, @since  // ドキュメントアノテーション
```

### 3. 現在の制限事項

**更新（v1.9.0）：** コア関数レベルアノテーション（`@inline`、`@noinline`、`@hot`、`@cold`、`@pure`、`@const`、`@flatten`、`@optimize`、`@warn_unused`、`@section`）はコンパイラバックエンドで完全に実装されました。以下のフェーズ3-5のループアノテーションとメモリアノテーションの提案はまだ未実装です。

## 提案されるアノテーション

### カテゴリ1：最適化ヒント

#### `@unroll(count?: number)`
**ターゲット：** ループ（for、while）
パフォーマンス重要なタイトループのループアンロールを提案。

#### `@simd` / `@nosimd`
**ターゲット：** 関数、ループ
SIMD ベクトル化の有効化または無効化。

#### `@likely` / `@unlikely`
**ターゲット：** if文、条件式
ホットパスの分岐予測ヒント。

#### `@const`
**ターゲット：** 関数
関数は同じ入力に対して常に同じ結果を返す（`@pure`より強い）。

#### `@tail_call`
**ターゲット：** 関数呼び出し
末尾呼び出し最適化（TCO）を要求。

#### `@flatten`
**ターゲット：** 関数
この関数内のすべての呼び出しをインライン化。

#### `@optimize(level: string)`
**ターゲット：** 関数
特定の関数のグローバル最適化レベルをオーバーライド。

### カテゴリ2：メモリ管理

#### `@stack`
**ターゲット：** 変数（配列、バッファ）
可能な場合はヒープの代わりにスタックに割り当て。

#### `@noalias`
**ターゲット：** 関数パラメータ（ポインタ/バッファ）
ポインタが他のポインタとエイリアスしないことを約束。

#### `@aligned(bytes: number)`
**ターゲット：** 変数（ポインタ、バッファ）、関数の戻り値
メモリアライメント要件を指定。

### カテゴリ3：コード生成制御

#### `@extern(name?: string, abi?: string)`
**ターゲット：** 関数
外部リンケージまたはFFIエクスポート用にマーク。

#### `@section(name: string)`
**ターゲット：** 関数、グローバル変数
シンボルを特定のELF/Mach-Oセクションに配置。

### カテゴリ4：エラーチェック

#### `@bounds_check` / `@no_bounds_check`
**ターゲット：** 配列/バッファ操作、ループ
グローバルの境界チェックポリシーをオーバーライド。

#### `@warn_unused`
**ターゲット：** 関数の戻り値
呼び出し側が戻り値を無視した場合に警告。

### カテゴリ5：FFI/相互運用

#### `@packed`
**ターゲット：** 型定義（define）
パディングなしのパック構造体を作成（C相互運用用）。

## 例

### 例1：高性能ベクトル数学

```hemlock
@simd
@flatten
fn vector_add(a: buffer, b: buffer, result: buffer, n: i32) {
    @unroll(8)
    for (let i = 0; i < n; i++) {
        let av = ptr_read_f64(a, i * 8);
        let bv = ptr_read_f64(b, i * 8);
        ptr_write_f64(result, i * 8, av + bv);
    }
}
```

### 例2：キャッシュ最適化データ構造

```hemlock
@packed
define CacheLineNode {
    next: ptr,
    data: i64,
    timestamp: u64,
    flags: u32,
    padding: u32
}
```

### 例3：再帰的末尾呼び出し最適化

```hemlock
@tail_call
fn sum_range(start: i32, end: i32, acc: i32): i32 {
    if (start > end) {
        return acc;
    }
    @tail_call
    return sum_range(start + 1, end, acc + start);
}
```

## 結論

この提案はHemlockに**15個の新しいコンパイラヘルパーアノテーション**を追加し、言語の「暗黙より明示」の哲学を維持しながら、開発者に明示的な最適化ヒントの提供を可能にします。

**主な利点：**

1. **パフォーマンス：** SIMD、アンロール、インライン化によるクリティカルパスの2-10倍高速化
2. **制御：** 開発者がデフォルトのコンパイラヒューリスティックをオーバーライド可能
3. **相互運用：** @extern、@packed、@alignedによるFFIサポートの向上
4. **安全性：** 明示的な@bounds_check/@no_bounds_checkで安全性のトレードオフが可視化
5. **明示的：** Hemlockの哲学に適合 - マジックなし、明確なディレクティブのみ

## 付録：完全なアノテーションリファレンステーブル

| アノテーション | ターゲット | 引数 | 説明 | C属性 |
|------------|--------|------|-------------|-------------|
| `@inline` | fn | 0 | インライン化を強制 | `always_inline` |
| `@noinline` | fn | 0 | インライン化を防止 | `noinline` |
| `@cold` | fn | 0 | まれに実行 | `cold` |
| `@hot` | fn | 0 | 頻繁に実行 | `hot` |
| `@pure` | fn | 0 | 副作用なし、グローバル読み取り可能 | `pure` |
| `@const` | fn | 0 | 副作用なし、グローバル読み取りなし | `const` |
| `@flatten` | fn | 0 | 関数内のすべての呼び出しをインライン化 | `flatten` |
| `@tail_call` | fn | 0 | 末尾呼び出し最適化を要求 | カスタム |
| `@optimize(level)` | fn | 1 | 最適化レベルをオーバーライド | `optimize("OX")` |
| `@unroll(factor?)` | loop | 0-1 | ループアンロールヒント | `#pragma unroll` |
| `@simd` | fn, loop | 0 | SIMDベクトル化を有効化 | `#pragma omp simd` |
| `@nosimd` | fn, loop | 0 | SIMDを無効化 | カスタム |
| `@likely` | if | 0 | 分岐が取られる可能性が高い | `__builtin_expect` |
| `@unlikely` | if | 0 | 分岐が取られる可能性が低い | `__builtin_expect` |
| `@stack` | let | 0 | スタック割り当て | カスタム |
| `@noalias` | param | 0 | ポインタエイリアスなし | `noalias` |
| `@aligned(N)` | let, fn | 1 | メモリアライメント | `aligned(N)` |
| `@extern(name?, abi?)` | fn | 0-2 | 外部リンケージ | `extern "C"` |
| `@section(name)` | fn, let | 1 | 特定セクションに配置 | `section("X")` |
| `@bounds_check` | fn | 0 | 境界チェックを強制 | カスタム |
| `@no_bounds_check` | fn | 0 | 境界チェックを無効化 | カスタム |
| `@warn_unused` | fn | 0 | 未使用の戻り値に警告 | `warn_unused_result` |
| `@packed` | define | 0 | 構造体パディングなし | `packed` |
