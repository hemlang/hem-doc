# Добро пожаловать в Hemlock

> "Маленький, небезопасный язык для безопасного написания небезопасных вещей."

**Hemlock** — это системный скриптовый язык, сочетающий мощь C с современной эргономикой скриптовых языков. Он включает ручное управление памятью, явный контроль и встроенную структурированную асинхронную конкурентность.

## Что такое Hemlock?

Hemlock создан для программистов, которым нужны:

- **Явный контроль** над памятью и выполнением
- **C-подобный синтаксис** с современными удобствами
- **Никакого скрытого поведения** или магии
- **Настоящая параллельная асинхронность** с конкурентностью на основе pthread

Hemlock — это НЕ язык с безопасной памятью и сборкой мусора. Вместо этого он даёт вам инструменты для безопасности (`buffer`, аннотации типов, проверка границ), не заставляя вас их использовать (`ptr`, ручное управление памятью, небезопасные операции).

## Быстрый пример

```hemlock
// Привет, Hemlock!
fn greet(name: string): string {
    return `Привет, ${name}!`;
}

let message = greet("Мир");
print(message);

// Ручное управление памятью
let buf = buffer(64);
buf[0] = 72;  // 'H'
buf[1] = 105; // 'i'
print(buf);
free(buf);
```

## Возможности на первый взгляд

| Возможность | Описание |
|-------------|----------|
| **Система типов** | i8-i64, u8-u64, f32/f64, bool, string, rune, ptr, buffer, array, object |
| **Память** | Ручное управление с `alloc()`, `buffer()`, `free()` |
| **Асинхронность** | Встроенные `async`/`await` с настоящим параллелизмом pthread |
| **FFI** | Прямой вызов функций C из разделяемых библиотек |
| **Стандартная библиотека** | 40 модулей, включая crypto, http, sqlite, json и другие |

## Пакетный менеджер

Hemlock поставляется с **hpm**, пакетным менеджером для управления зависимостями:

```bash
hpm init my-project
hpm add some-package
hpm run
```

Подробности смотрите в разделах документации hpm.
